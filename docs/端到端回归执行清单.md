# 端到端回归执行清单（报名 -> 审核 -> 导出）

最后更新：2026-05-01

## 1. 执行目标

在发布前验证核心链路可用：

1. 学生提交报名
2. 管理员审核通过/驳回
3. 管理端导出 `CSV/XLSX`

## 2. 用例清单

| 编号 | 用例 | 预期结果 |
| --- | --- | --- |
| E2E-01 | 学生提交个人报名（含附件） | 报名创建成功，状态为 `submitted` |
| E2E-02 | 同一学生重复提交同一比赛 | 被阻止，返回“已存在有效报名” |
| E2E-03 | 管理员单条审核通过 | 状态变为 `approved`，有审核日志 |
| E2E-04 | 管理员批量驳回 | 返回更新条数，状态批量更新 |
| E2E-05 | 导出 CSV | 文件可下载，列头完整 |
| E2E-06 | 导出 XLSX | 文件可下载，含“报名汇总/审核日志”两张表 |
| E2E-07 | 比赛作用域越权审核 | 返回 403 或业务拒绝 |

## 4. 最低通过门槛

1. E2E-01 ~ E2E-06 必须全部通过。
2. E2E-07 必须确认被拒绝（fail-closed）。
3. 若任一失败，不得进入正式发布。
