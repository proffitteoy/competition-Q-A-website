本目录不再作为通用提示词模板集合使用，当前阶段改为本项目的产品与实施文档入口。

项目目标：

**围绕学院竞赛场景，建立从发布、咨询、报名、审核、导出到资料沉淀的一整套闭环平台。**

当前优先级不是直接做完整系统，而是先固定以下内容：

1. 页面清单
2. 每页信息架构
3. 用户操作路径
4. 关键数据状态
5. 管理员工作流
6. 视觉风格规范
7. Mock 数据版前端范围

## 当前结论

- 平台定位不是泛论坛，也不是普通官网，而是“学院竞赛管理与问答平台”。
- 业务核心优先级为：`报名与审核` > `竞赛门户与通知` > `围绕竞赛的问答沉淀` > `精细化权限与统计扩展`。
- 第一版应先完成公开页、报名链路、管理员审核与导出，不应一开始就铺开社区化功能。
- 问答系统是服务竞赛的辅助模块，不应主导首页结构和信息层级。
- 技术路线优先选择 `Next.js + Tailwind CSS + shadcn/ui`。
- 模板策略为：`ShadcnStore Dashboard + Landing Template` 做主骨架，`Launch UI` 只做学院竞赛门户首页与展示区补强。
- 组件策略为：`shadcn/ui` 做底座，业务表格/表单优先复用主模板现成依赖，展示层优先参考本地 `Launch UI`，不堆多套完整 UI 体系。
- 阶段 2 的默认后端路线已固定为：`Next.js 单体 + PostgreSQL + Drizzle + Auth.js + S3 兼容对象存储`。
- 当前代码骨架已落到仓库根目录的 `src/` 与 `public/`，外部开源目录转为上游来源和待清理对象。
- 根项目路由已完成首轮去模板化清理，主业务路由聚焦竞赛门户、报名、登录和管理员工作台，同时保留认证与错误页基础路由。
