# 问题修复记录（2026-05-01）

## 背景

本次修复针对以下 5 个线上可见问题：

1. 报名审核列表中的“提交时间”显示错误
2. 问答发布时出现生产态 `Server Components render` 红字报错
3. 从比赛列表点击“通知公告 / 报名资料 / 名人堂”会回到首页但无法正确定位
4. 已登录状态下，部分页面顶栏仍显示“登录”
5. 不上传附件可以报名，上传附件后报 `ENOENT: no such file or directory, mkdir '/var/task/public/uploads'`

## 根因与修复

### 1. 报名审核提交时间错误

- 根因：
  - 报名、通知、比赛时间窗口的格式化直接使用了服务器本地时区或 `toISOString()`。
  - 部署环境在 UTC 时，会把东八区时间显示早 8 小时，后台审核列表最明显。
- 修复：
  - 新增 [src/lib/date-time.ts](/F:/竞赛问答平台/src/lib/date-time.ts)。
  - 将报名审核、审核日志、通知时间、比赛时间窗口统一改为按 `APP_TIME_ZONE -> TZ -> Asia/Shanghai` 格式化。
  - 修改文件：
    - [src/server/repositories/application-repository.ts](/F:/竞赛问答平台/src/server/repositories/application-repository.ts)
    - [src/server/repositories/notice-repository.ts](/F:/竞赛问答平台/src/server/repositories/notice-repository.ts)
    - [src/server/repositories/competition-repository.ts](/F:/竞赛问答平台/src/server/repositories/competition-repository.ts)

### 2. 问答发布时报生产态红字

- 根因：
  - “我要提问”原先直接调用 Server Action。
  - 一旦服务端抛错，生产环境只会把错误压成通用 digest，前端看起来像“系统炸了”，无法拿到真实业务错误。
- 修复：
  - 新增 [src/app/api/questions/route.ts](/F:/竞赛问答平台/src/app/api/questions/route.ts)，改为通过 Route Handler 提交问题。
  - [src/components/questions/ask-question-form.tsx](/F:/竞赛问答平台/src/components/questions/ask-question-form.tsx) 改为 `fetch("/api/questions")` 提交，并在成功后 `router.refresh()` 刷新问题列表。
  - 这样即使服务端返回业务错误，也能明确显示错误信息，不再出现生产态 digest 红字。

### 3. 比赛列表页顶部其他导航会回首页但不定位

- 根因：
  - 顶栏中的“通知公告 / 报名资料 / 名人堂”本质是首页锚点。
  - 在客户端路由之间跳转时，`Link` 对 `/#anchor` 的处理不稳定，表现为只回到首页顶部，哈希定位丢失。
- 修复：
  - [src/components/marketing/portal-navbar.tsx](/F:/竞赛问答平台/src/components/marketing/portal-navbar.tsx) 中把这类锚点链接改为原生 `<a href="/#...">`。
  - 保留“比赛列表”等普通路由继续使用 `Link`。

### 4. 已登录但仍显示“登录”

- 根因：
  - 顶栏是客户端组件，部分页面没有显式把当前用户传进去。
  - 这些页面会把“未传入用户”误当作“未登录”，尤其是比赛列表页和报名页。
- 修复：
  - [src/components/marketing/portal-navbar.tsx](/F:/竞赛问答平台/src/components/marketing/portal-navbar.tsx) 增加客户端会话兜底：
    - 如果页面没传 `currentUser`，顶栏会主动调用 `getSession()` 读取当前登录态。
  - [src/app/competitions/[id]/apply/page.tsx](/F:/竞赛问答平台/src/app/competitions/[id]/apply/page.tsx) 改为服务端显式传入当前用户，避免报名页首屏误显示“登录”。
  - [src/app/me/applications/page.tsx](/F:/竞赛问答平台/src/app/me/applications/page.tsx) 也补传当前用户，保证“我的报名”页顶栏一致。

### 5. 上传附件时报 `/var/task/public/uploads` 不存在

- 根因：
  - 本地上传驱动默认写入 `public/uploads`。
  - 在 Serverless / 只读部署环境中，`/var/task/public` 不可写，`mkdir` 直接失败。
- 修复：
  - [src/lib/storage/local-storage.ts](/F:/竞赛问答平台/src/lib/storage/local-storage.ts) 调整本地驱动策略：
    - 开发环境默认写入仓库 `.uploads`
    - 生产环境默认写入系统临时目录 `os.tmpdir()`
  - 不再依赖 `public/uploads` 作为可写目录。
  - 同时为本地驱动补了 `/api/uploads?key=...` 读取入口：
    - [src/app/api/uploads/route.ts](/F:/竞赛问答平台/src/app/api/uploads/route.ts)
  - 上传返回的 `publicUrl` 现在指向可读取的 API 地址，而不是假定静态目录可直接暴露。
