# 当前已实现功能清单（MVP进行中）

最后更新：2026-04-30

## 1. 认证与会话
- 已接入 `Auth.js + Credentials` 登录。
- 登录页已调用真实 `signIn`，支持 `callbackUrl` 回跳。
- 注册页已调用 `POST /api/auth/register`，注册成功后自动尝试登录。
- 已提供统一会话读取：`src/lib/auth/session.ts`。

## 2. 路由守卫
- `/admin/*` 已在服务端 Layout 做鉴权守卫。
- 未登录用户会被重定向到 `/sign-in`。
- 非管理员角色会被重定向到 `/errors/forbidden`。

## 3. 数据层底座
- 已完成 `PostgreSQL + Drizzle ORM + drizzle-kit` 基础接入。
- 已落地核心表：用户、角色分配、比赛、报名、审计日志、Auth.js 相关表。
- 已提供脚本：`db:generate`、`db:migrate`、`db:migrate:sql`、`db:push`、`db:seed`。

## 4. 学生报名链路
- 报名表单已提交真实数据（含 `studentId/phone/email/statement/teamName`）。
- 新增上传接口：`POST /api/uploads`（`multipart/form-data`）。
- 上传策略已实现：扩展名白名单、单文件大小限制、总大小限制。
- 当前上传存储为本地目录（默认 `public/uploads`）。
- 报名写入已支持 `attachments` 元数据快照。

## 5. 后台比赛管理
- 新增 `GET/POST /api/admin/competitions`。

- 比赛状态变更已接入状态流转服务与权限校验。
- `/admin/competitions` 已接入真实数据，支持新建、编辑、删除。

## 6. 后台用户与角色管理
- 已有 `GET /api/admin/users`、`PATCH /api/admin/users/[id]`。
- 支持用户状态更新。

- `/admin/users` 已接入真实数据与编辑操作。

## 7. 后台报名审核
