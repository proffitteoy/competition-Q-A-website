# 当前已实现功能清单

> 统计范围：当前仓库 `src/` 代码与已落地脚本（阶段 2 进行中）

## 1. 路由与页面骨架

已具备公开端、认证页与后台主路由骨架，并可运行：

- 公开端：`/`、`/competitions`、`/competitions/[id]`、`/competitions/[id]/apply`、`/me/applications`
- 认证页：`/sign-in`、`/sign-up`、`/forgot-password`
- 后台：`/admin`、`/admin/competitions`、`/admin/applications`、`/admin/notices`、`/admin/users`、`/admin/schedule`

## 2. 数据层基础工程

已完成 `PostgreSQL + Drizzle` 基础工程接入：

- 数据库配置与客户端：
  - `src/lib/db/config.ts`
  - `src/lib/db/client.ts`
- Schema 已覆盖核心业务对象：
  - `user`、`role_assignment`
  - `competition`、`competition_notice`、`competition_attachment`、`competition_faq`、`competition_status_log`
  - `registration_form`、`registration_form_version`、`registration_form_field`
  - `registration`、`registration_team_member`、`registration_revision`、`registration_audit_log`
  - Auth.js 相关表：`auth_account`、`auth_session`、`auth_verification_token`
- 迁移与种子脚本：
  - `drizzle.config.ts`
  - `src/lib/db/migrations/0000_init.sql`
  - `scripts/apply-migration.mjs`
  - `scripts/seed-db.mjs`

## 3. 认证与会话

已落地 Auth.js 后端骨架：

- `Credentials` 登录提供器
- 数据库 session 方案
- `next-auth` 路由：`/api/auth/[...nextauth]`
- 服务层统一会话入口：`src/lib/auth/session.ts`
  - 优先读 Auth.js session
  - 数据库/认证未就绪时支持环境变量兜底会话（开发态）

## 4. 权限与状态规则（服务层）

已实现并集中到服务层：

- 比赛作用域权限校验（全局角色 + 比赛级作用域）
- 报名状态流转规则（含 `cancelled`）
- 比赛状态流转规则
- 比赛异常终止联动：
  - `registration_open -> archived` 时，active 报名批量转 `cancelled`
  - 同步写入 `registration_audit_log`

## 5. 报名主链路

学生端主链路已可用：

- 学生提交报名（入库 + 快照 + revision + audit log）
- 我的报名查询
- 重复报名校验（active 状态集合约束）
- 申请人快照写入与回读

相关入口：

- `POST /api/applications`
- `GET /api/me/applications`
- Server Actions：`src/actions/applications.ts`

## 6. 管理员审核链路

管理员单条与批量审核已可用：

- 单条审核：通过 / 驳回 / 撤回 / 取消
- 批量处理：通过 / 驳回 / 取消
- 审核日志入库
- 后台筛选：关键词、状态、比赛

相关入口：

- `GET /api/admin/applications`
- `POST /api/admin/applications`（批量处理）
- `POST /api/admin/applications/[id]/review`（单条处理）

## 7. 导出能力

已实现管理员导出：

- `CSV` 导出
- `XLSX` 导出（报名汇总 + 审核日志）
- 路由：`GET /api/admin/applications/export?format=csv|xlsx`

## 8. 公开端数据来源切换

以下页面已从 mock 常量切到仓储读取：

- 首页 `/`
- 比赛列表 `/competitions`
- 比赛详情 `/competitions/[id]`
- 赛程页 `/admin/schedule`
