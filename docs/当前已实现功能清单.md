# 当前已实现功能清单（MVP进行中）

最后更新：2026-05-01

## 1. 认证与会话
- 已接入 `Auth.js + Credentials` 登录。
- 登录页已调用真实 `signIn`，支持 `callbackUrl` 回跳。
- 注册页已调用 `POST /api/auth/register`，注册成功后自动尝试登录。
- 已提供统一会话读取：`src/lib/auth/session.ts`。
- 已增加生产环境安全门禁：`AUTH_SECRET` 长度检查、会话有效期可配置、默认收紧环境变量会话回退。

## 2. 路由守卫
- `/admin/*` 已在服务端 Layout 做鉴权守卫。
- 未登录用户会被重定向到 `/sign-in`。
- 非管理员角色会被重定向到 `/errors/forbidden`。

## 3. 数据层底座
- 已完成 `PostgreSQL + Drizzle ORM + drizzle-kit` 基础接入。
- 已落地核心表：用户、角色分配、比赛、报名、审计日志、Auth.js 相关表。
- 已提供脚本：`db:generate`、`db:migrate`、`db:migrate:sql`、`db:push`、`db:seed`、`seed:debug`。

## 4. 学生报名链路
- 报名表单已提交真实数据（含 `studentId/phone/email/statement/teamName`）。
- 新增上传接口：`POST /api/uploads`（`multipart/form-data`）。
- 上传策略已实现：扩展名白名单、单文件大小限制、总大小限制。
- 当前上传存储为本地目录（默认 `public/uploads`）。
- 报名写入已支持 `attachments` 元数据快照。
- 报名表单已支持本地草稿暂存与自动恢复（不含附件二进制）。

## 5. 后台比赛管理
- 新增 `GET/POST /api/admin/competitions`。
- 新增 `GET/PATCH/DELETE /api/admin/competitions/[id]`。
- 比赛状态变更已接入状态流转服务与权限校验。
- `/admin/competitions` 已接入真实数据，支持新建、编辑、删除。

## 6. 后台用户与角色管理
- 已有 `GET /api/admin/users`、`PATCH /api/admin/users/[id]`。
- 支持用户状态更新。
- 支持主角色更新与比赛作用域绑定（`competition_admin/content_editor`）。
- `/admin/users` 已接入真实数据与编辑操作。

## 7. 后台报名审核
- `/admin/applications` 已使用真实数据源。
- 支持筛选、单条审核、批量审核。
- 支持审计日志写入。
- 支持导出 `CSV/XLSX`。

## 8. 后台通知与仪表盘
- 通知已支持 `GET/POST/PATCH/DELETE` API。
- `/admin/notices` 已接入真实数据、富文本编辑、草稿暂存、状态管理。
- `/admin` 仪表盘已接入 `GET /api/admin/dashboard` 真实聚合数据，并新增待审核数、本周新增、通过率与状态分布。

## 9. 生产化与回归
- 上传链路已支持存储驱动切换：`UPLOAD_STORAGE_DRIVER=local|s3`，并支持 `S3/OSS` 兼容配置。
- 后台关键接口已收紧比赛作用域策略（fail-closed）。
- 新增权限回归脚本：`npm run test:permissions`。
- 新增调试管理员脚本：`npm run admin:create-debug`（写入/更新 `super_admin` 账号）。
- `seed:debug` 已补充扩展样例：多比赛状态、多通知状态、多报名状态，便于后台页面联调。
- 新增文档：
  - `docs/生产数据库迁移与回滚手册.md`
  - `docs/端到端回归执行清单.md`

## 10. 构建与质量状态
- `npm run build`：通过（2026-04-30）。
- `npm run lint`：通过（仅剩 TanStack Table 的 React Compiler 兼容性 warning）。
