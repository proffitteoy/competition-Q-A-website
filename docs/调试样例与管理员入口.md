# 调试样例与管理员入口

最后更新：2026-05-01

## 1. 一键写入调试样例

在已配置数据库的前提下，执行：

```bash
npm run db:migrate
npm run db:seed
```

`db:seed` 会清空并重建核心表数据，请不要在生产库执行。

## 2. 已内置的调试样例范围

种子数据覆盖以下场景：

1. 比赛状态：`draft`、`upcoming`、`registration_open`、`in_progress`、`finished`、`archived`
2. 通知状态：`published`、`draft`、`withdrawn`
3. 报名状态：`submitted`、`approved`、`rejected`、`withdrawn`、`cancelled`
4. 角色样例：`super_admin`、`competition_admin`、`content_editor`、`student_user`
5. 比赛附加信息：FAQ、附件、时间线、亮点字段

## 3. 调试账号

### 3.1 种子管理员账号（来自 `db:seed`）

默认管理员邮箱：

- `super@college.example`（`super_admin`）

默认密码：

- 取环境变量 `MVP_BOOTSTRAP_SUPER_ADMIN_PASSWORD`
- 若未设置，默认 `DebugAdmin2026!`

### 3.2 临时创建/刷新管理员账号

```bash
npm run admin:create-debug
```

默认创建：

- `debug.admin@college.example`
- 密码：`DebugAdmin2026!`
- 角色：`super_admin`

可通过环境变量覆盖：

- `DEBUG_ADMIN_EMAIL`
- `DEBUG_ADMIN_PASSWORD`
- `DEBUG_ADMIN_NAME`

## 4. 管理员界面入口

1. 打开登录页：`/sign-in`
2. 使用管理员账号登录
3. 登录成功后进入：`/admin`

常用后台路由：

1. `/admin/competitions`：比赛管理
2. `/admin/applications`：报名审核与导出
3. `/admin/notices`：通知管理（含富文本）
4. `/admin/users`：用户与角色管理
5. `/admin/schedule`：赛程时间线视图

## 5. 无数据库时本地调试（仅临时）

可用环境变量兜底会话：

```env
MVP_ALLOW_ENV_SESSION_FALLBACK=true
MVP_SESSION_USER=本地管理员
MVP_SESSION_ROLE=super_admin
```

重启 `dev` 后可访问 `/admin`。  
注意：这种方式不写入真实数据库，只适合界面联调。
